# Compass — Claude Guidelines

See README.md for project structure and make targets.

## Key Packages

- `lib/auth` — JWT/Keycloak validation; exports `JWTValidator` and `KeycloakClient` interfaces plus gRPC/HTTP interceptors
- `lib/flags` — shared `cli.Flag` definitions (Postgres URL, Redis URL, gRPC addr, …)
- `lib/middleware` — gRPC middleware (last-resort error/panic handling)
- `lib/oops` — domain error helpers (`oops.Internal`, `oops.NotFound`, …)
- `lib/quote` — quote domain: `Model` interface, LRU cache, Postgres store
- `services/api` — AuthService (Login via Keycloak)
- `services/quote` — QuoteService (exchange, OHLCV, security)
- `postgres/gen/`, `protocols/gen/` — generated code, do not edit manually

## Imports

Three groups separated by blank lines: stdlib → third-party → internal.
Use package aliases for clarity: `pb` for protobuf, `quoteLib` when `lib/quote` would collide with the local package name.

## Error Handling

- **Service layer** (gRPC handlers): validate inputs with `status.Error(codes.X, "...")` directly; pass model errors up as-is (`return err`).
- **Model/store layer**: wrap all errors with `lib/oops` — `oops.Internal(err)` for unexpected/DB errors, `oops.NotFound(...)` / `oops.InvalidArgument(...)` etc. for domain errors. Never use bare `errors.New` sentinel values for domain errors.
- The gRPC middleware in `lib/middleware` is the last-resort safety net; handlers do not need defensive wrapping.

## Authentication

- `lib/auth.GRPCUnaryInterceptor` / `GRPCStreamInterceptor` validate `Authorization: Bearer <token>` from gRPC metadata and inject the user ID into `ctx` via `auth.UserIDKey`.
- Use `auth.GetUserIDFromContext(ctx)` in handlers to retrieve it; return `codes.Unauthenticated` if missing.
- Methods that don't require auth (e.g. Login) are passed as `ignoreMethods` to the interceptors in `main.go`.

## Logging

- Use `zerolog` via context: `log := zerolog.Ctx(ctx)`.
- The logger is injected into context at startup in `main.go`; propagate it via `ctx`, never as a parameter.
- Log at `Warn` for recoverable anomalies, `Error` for unhandled failures. No `Info`-level noise inside library code.

## Interfaces & Types

- Interfaces are defined in the consumer's package (e.g., `Model` lives in `lib/quote`).
- Implementations are unexported structs (`store`, `lruCache`, `serverStream`).
- Constructor naming: `New(...)` for service/public types, `newX(...)` for unexported ones.

## Comments

- Package doc comment on non-trivial packages.
- Doc comments on all exported types, interfaces, and functions.
- Comments on interface methods only when the error contract or non-obvious behaviour needs documenting.
- No comments on self-evident code.

## Testing

- Table-driven tests with `tests []struct{ name string; ... }` and `t.Run(tc.name, ...)`.
- Helper functions call `t.Helper()`.
- Mocks generated by `mockery`; use `EXPECT()` for strict expectations, `.On(...)` for loose stubs.
- `require` (testify) for fatal setup assertions; plain `t.Error` / `t.Errorf` for individual field checks.
- Service tests call `New(m).Method(...)` inline — no intermediate variable for the service.
- Mock stream types are defined in the test file that uses them.
